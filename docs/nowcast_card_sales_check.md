# Nowcast 카드매출 대체지표 확인 결과

확인일: 2026-06-23

## 1. 결론

통계데이터센터 Nowcast의 `가맹점 카드매출액`과 `신용카드 이용금액`은 부산 시도 단위 주간 데이터로 수집 가능하다.

다만 정식 공공 OpenAPI 문서가 있는 방식이 아니라 Nowcast 화면에서 사용하는 Ajax JSON 엔드포인트를 호출하는 구조다. 따라서 운영 자동화는 가능하지만, URL/파라미터 변경 가능성을 감안해 수집 실패 알림과 fallback 처리가 필요하다.

## 3. 확인된 호출 방식

기본 페이지:

```text
https://data.mods.go.kr/nowcast/main.do?initId=6
```

지표명 조회:

```http
POST https://data.mods.go.kr/nowcast/selectIndcrTitle.do
indcr_id=6
```

부산 시계열 조회:

```http
POST https://data.mods.go.kr/nowcast/listIndcrDataAjax.do
indcr_id=6
val1=21
wklId=52
mode=
initId=6
```

지역별 시도 시계열 조회:

```http
POST https://data.mods.go.kr/nowcast/listIndcrAreaDataAjax.do
indcr_id=6
val1=
dpth=1
mode=
wklId=52
initId=6
```

## 4. 비교시점 파라미터

| `wklId` | 의미 | 화면 표기 |
|---:|---|---|
| `0` 또는 공백 | 2020년 1월 대비 | 2020년 1월 |
| `1` | 전주 대비 | 지난주 |
| `4` | 4주 전 대비 | 4주전 |
| `52` | 52주 전 대비 | 52주전 |

대시보드에는 `52주전 대비`를 기본값으로 쓰는 것을 권장한다. `2020년 1월 대비`는 코로나 초기 기준점이라 2026년 이후 민생경제 변화를 설명하는 데 부적절할 수 있다.

## 5. 부산 최신값 확인

2026-06-23 호출 기준:

| 지표 | 기준일 | 2020년 1월 대비 | 전주 대비 | 4주 전 대비 | 52주 전 대비 |
|---|---|---:|---:|---:|---:|
| 가맹점 카드매출액 | 2026-06-05 | `+4.1%` | `-12.8%` | `-6.8%` | `+5.8%` |
| 신용카드 이용금액 | 2026-06-05 | `+6.4%` | `-18.4%` | `+1.8%` | `+9.3%` |

주의:

- 응답 원값은 소수 비율이고, 화면 표시는 `* 100` 후 `%`로 변환한다.
- 날짜는 Unix epoch millisecond로 내려오며, KST 기준 날짜로 변환한다.
- 일부 화면 코드에서는 차트 표시 시 `BASE_DT + 1일` 처리를 한다. 운영 DB에는 원 응답 기준일과 화면 표시일을 구분해 저장하는 것이 안전하다.

## 6. 운영 판단

| 항목 | 판단 |
|---|---|
| 부산 지역 단위 | 가능 |
| 최신성 | 주간, 약 2~3주 지연 |
| 자동화 | 가능. 단, 화면용 Ajax 엔드포인트 기반 |
| 원 단위 매출액 | 불가. 공개 화면/JSON 응답은 변동률만 제공 |
| 대시보드 채택 | 권장 |

표출명 권장:

- `부산 가맹점 카드매출액 변동률`
- `부산 신용카드 이용금액 변동률`

표출 주석:

> 출처: 국가데이터처 통계데이터센터 Nowcast. 민간 카드 거래정보 기반 속보성 지표이며, 원 단위 매출액이 아니라 기준시점 대비 변동률이다.

## 7. 금액 원자료 확인 여부

Nowcast의 `신용카드 이용금액`과 `가맹점 카드매출액`은 지표명에 금액/매출액이 포함되지만, 현재 공개 화면에서 내려오는 데이터 필드는 다음으로 제한된다.

```json
{
  "BASE_DT": 1780585200000,
  "INDCR_VL": 0.09261017
}
```

확인한 엔드포인트:

- `listIndcrDataAjax.do`
- `listIndcrAreaDataAjax.do`
- `listIndcrClsDataAjax.do`

세 엔드포인트 모두 `INDCR_VL`만 제공하며, 원 단위 이용금액 또는 매출액 필드는 확인되지 않았다.

따라서 Nowcast 기준으로는 부산 신용카드 이용금액의 원 단위 금액을 확인할 수 없고, `52주전 대비 +9.3%` 같은 변동률 지표로만 사용할 수 있다.
