# 민생100일 지표 대시보드 구현 정리

작성일: 2026-06-23

## 1. 목적

부산 민생100일 대시보드에서 경제지표 변화를 보여주기 위한 별도 서비스다. 기존 공공계약 모니터링시스템과 지역상품추천서비스가 운영 중인 네이버클라우드 서버를 사용하되, 코드/DB/가상환경/systemd/nginx route를 분리한다.

## 2. 배포 원칙

| 항목 | 기준 |
|---|---|
| 서버 경로 | `/opt/minsaeng100` |
| 실행 계정 | `minsaeng` 별도 계정 권장 |
| API 포트 | `127.0.0.1:8010` |
| 대시보드 포트 | `127.0.0.1:8510` |
| API 서비스 | `minsaeng100-api.service` |
| 화면 서비스 | `minsaeng100-dashboard.service` |
| nginx route | `/minsaeng100/`, `/minsaeng100-api/` |
| DB | `/opt/minsaeng100/data/minsaeng100.sqlite` |

제약:

- 기존 `8000`, `8001`, `8501`, `8502` 포트와 충돌하지 않도록 한다.
- 기존 `/opt/busan`, `/opt/advisor` 내부 파일을 수정하지 않는다.
- 실서버 적용 전 `df -h`, `free -h`, `ss -ltnp`, `systemctl`, `nginx -T`, cron 구성을 확인한다.

- 카드매출 자료는 카드사 가맹점 결제 승인 기반의 민간구매데이터다.

- 공개 갱신주기가 연간이므로 민생100일의 최신 월별 운영지표로 쓰지 않는다.
- 필요하면 과거 상권 소비 구조 분석 또는 기준선 분석에만 사용한다.
- 전체 카드시장 또는 부산 전체 민간소비를 대표한다고 표현하면 안 된다.

운영 판단:

- 금액 지표가 필요하면 부산시 Big-데이터웨이브 상황판의 `card_spend`를 우선 후보로 둔다.
- 2026-06-23 확인 결과, 상황판 iframe `https://bs.webasp.kr/dashboard`는 `GET /api/dashboard/eap?region=26&date=YYYYMM`로 부산 생활데이터를 조회한다.
- `GET /api/dateRangeInfo?region=26` 기준 부산 수록범위는 `2023-06~2026-04`다. 2026-04 `card_spend.current`는 `82,933,157,226원`이며, 전월 대비 `+3.81%`, 전년 대비 `+12.14%`다.
- 상황판 매뉴얼 기준 소비패턴 신용카드 데이터의 자료출처는 `BC카드`, 단위는 `원`, 데이터레벨은 `시도·시군구·행정동`이다.
- 단, 이 경로는 공공 OpenAPI가 아니라 화면용 API다. 민생100일 운영 서비스에서 자동 수집하려면 부산시 또는 상황판 운영기관에 재사용 권한과 호출 제한을 확인해야 한다.
- 변동률 지표가 필요하면 Nowcast의 `가맹점 카드매출액`과 `신용카드 이용금액`을 병행 후보로 둔다.
- 2026-06-23 확인 결과, Nowcast `가맹점 카드매출액(indcr_id=6)`과 `신용카드 이용금액(indcr_id=1)`은 부산 코드 `21`로 주간 시계열 JSON 수집이 가능하다.
- 2026-06-23 확인 기준 최신값은 2026-06-05이며, 부산 가맹점 카드매출액은 52주전 대비 `+5.8%`, 부산 신용카드 이용금액은 52주전 대비 `+9.3%`다.
- Nowcast 값은 원 단위 매출액이 아니라 기준시점 대비 변동률이다. 대시보드 기본 기준시점은 `52주전 대비`를 권장한다.
- 한국관광 데이터랩은 월별 관광소비를 제공하므로, 관광상권·외부수요 설명에는 적합하다.
- 관광 데이터의 소매·유통 계열을 쓰더라도 `부산 전체 소매매출`이 아니라 `관광소비 중 소매·유통 소비`로 표기해야 한다.
- KDX/NH농협카드 자료는 일자별 부산 소비현황으로 유용하지만 공공 OpenAPI가 아니므로 운영 자동화는 계약·다운로드 조건 확인 후 결정한다.
- BECOS `개인신용카드사용액(idx=114)`은 금액 단위가 `백만원`인 월별 지표다. ECOS 공식 API 확인 결과 `601Y002`에서 부산 코드 `B`, 합계 `1000`, 총액 `TOT`로 조회 가능하다. 다만 수록 종료가 `2023-08`이므로 민생100일 운영 본편에서는 제외하고 과거 참고지표로만 사용한다.

비교 표출 기준:

- 기본값은 `부산 최신값`이다.
- 비교 1은 같은 기준기간의 `전국/전체 기준값`이다. Nowcast는 `val1` 공란 호출값을 `전국/전체` 비교값으로 저장한다.
- 비교 2는 같은 부산 지표의 `직전기간 값 대비 증감`이다. 월별 지표는 전월 대비, 주간 지표는 직전 주차 대비로 표시한다.
- 비율 지표의 비교 차이는 `%p`로 표시한다. 금액·건수 지표의 직전기간 증감은 증감률 `%`로 표시한다.
- 부산 Big-데이터웨이브 카드소비액은 전국값이 확인되지 않았으므로 전월 대비만 표출하고, 전국 비교는 Nowcast 카드 변동률로 대체한다.
- 소비자심리지수는 지역 월 표 `301/DT_511Y004`에서 부산값을, 전국 월 표 `301/DT_511Y002`에서 전국값을 수집한다.
- 소상공인·전통시장 BSI의 전국/전체 비교값은 별도 전국 집계가 아니라 `17개 시도 단순평균`이다.
- 부산 동행종합지수는 `202/DT_111_1`, 전국 경기동행종합지수는 `101/DT_1C8015`에서 수집한다. 서로 다른 통계표지만 모두 `2020=100` 동행종합지수이므로 같은 기준월 비교값으로 병행 표출한다.

주의:

- 한국관광 데이터랩은 신용카드 데이터 기반 해당지역 관광소비액을 제공한다.
- 내국인, 현지인, 외지인 구분이 가능하다.
- 데이터랩 안내상 빅데이터 방문자수 및 관광소비금액은 총량보다 추세 분석 용도로 활용하는 것이 권장된다.
- 지역별 관광 현황 데이터는 2020년 1월부터 제공되며, 월간 조회는 조회 시작일 기준 최대 18개월 제한이 있다.
- 관광소비는 부산시민의 일반 소비심리나 전체 소매매출이 아니라 관광 관련 소비 흐름이다. 민생경제 본편에서는 보조 패널로 배치한다.

## 4. 데이터 저장 기준

- 자동수집 지표는 `observations` 테이블에 `indicator_id`, `base_period`, `value`, `unit`, `source` 기준으로 저장한다.
- 수동 입력 지표는 별도 테이블에 저장한다.
- 금액은 원 단위로 저장하고 화면에서 억원 단위로 변환한다.
- 동일 기준월 재입력은 update로 처리하되, 운영 단계에서는 변경 이력 테이블을 추가해야 한다.
